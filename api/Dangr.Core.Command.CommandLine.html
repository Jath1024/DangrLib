Remarks

CommandLine is an object that will parse a given command line string, and break it down into its components which can be accessed as needed.

The Command Name is the first element of the command line string, and represents the command that should be run.

The arguments are parsed from the portion of the command line after the name. Arguments are seperated by whitespace and can be specified in one of three different ways.

  • Flags Arguments that begin with '-' and that are not followed by a value, are stored as flags. Flags are used to represent the boolean value true when present, and false otherwise.
  • Named arguments Arguments that begin with '-' and that are followed by a value, are stored as named arguments. Named arguments can be accessed as name/value pairs.
  • Positional arguments Values that do not begin with '-' and that are not preceded by an argument that does, are positional arguments. Positional arguments are stored as a list in the order they are specified.

Command and argument names can be any string of characters that do not contain one of the following values: `~!@#$%^&*()=+[]{}|\;:"',<>/?. They also cannot start with the '-' character. When reading flag or argument names, the '-' prefix will be removed from the name. Command names will always be converted to lowercase. Argument names are case sensitive.

Value arguments may be contained in single (') or double (") quotes, which can be escaped using a backslash () character. They cannot start with the '-' character, and any whitespace must be contained in quotes. Otherwise there are no restrictions on the characters in a value argument. When reading values, surrounding quote characters will be removed, and escaped quote characters will be replaced with their unescaped values.

Examples

Example 1
The following example will create a CommandLine that executes 'TestCommand' using named arguments:

CommandLine commandLine = new CommandLine(
    "TestCommand -Named1 argument1 -Named2 \"argument \\\"2\\\"\"");
Console.WriteLine($"CommandName: {commandLine.CommandName}");
Console.WriteLine($"Named1: {commandLine.NamedArguments["Named1"]}");
Console.WriteLine($"Named2: {commandLine.NamedArguments["Named2"]}");

Will output:

CommandName: TestCommand
Named1: argument1
Named2: argument "2"

Example 2
The following example will create a CommandLine that executes 'TestCommand' using flags:

CommandLine commandLine = new CommandLine(
    "TestCommand -Flag1 -Flag2 -Named1 arg");
Console.WriteLine($"CommandName: {commandLine.CommandName}");
Console.WriteLine($"Flag1: {commandLine.Flags.Contains("Flag1")}");
Console.WriteLine($"Flag2: {commandLine.Flags.Contains("Flag2")}");
Console.WriteLine($"Flag3: {commandLine.Flags.Contains("Flag3")}");
Console.WriteLine($"Named1: {commandLine.NamedArguments["Named1"]}");

Will output:

CommandName: TestCommand
Flag1: true
Flag2: true
Flag3: false
Named1: arg

Example 3
The following example will create a CommandLine that executes 'TestCommand' using positional arguments:

CommandLine commandLine = new CommandLine(
    "TestCommand positional0 \"positional arg 1\" -Named1 named positional2 -Flag1");
Console.WriteLine($"CommandName: {commandLine.CommandName}");
Console.WriteLine($"Named1: {commandLine.NamedArguments["Named1"]}");
Console.WriteLine($"Flag1: {commandLine.Flags.Contains("Flag1")}");
for(int i = 0; i < this.PositionalArguments.Count; i++)
{
    Console.WriteLine($"Positional {i}: {this.PositionalArguments[i]}");
}

Will output:

CommandName: TestCommand
Named1: named
Flag1: true
Positional 0: positional0
Positional 1: positional arg 1
Positional 2: positional2
