Remarks

The CommandExecutor is an object that will parse a given command line and execute it as a command. Different IDangrCommands can be registered.

CommandExecutors can also be nested to create a scope for specific commands. Commands registered with the child executor can only be accessed when executed with the child executor, but commands registered with the parent executor can be executed from the child. Names are resolved starting with the child, so any commands registered with the same name will override commands in the parent executor.

Command lines are parsed using a CommandLine instance and should follow those conventions for command and argument names.

Examples

Example 1
The following example will create a CommandExecutor, register a command called "Output" and execute it.

TextWriter outputStream = new StringWriter();
TextWriter errorStream = new StringWriter();

CommandExecutor executor = new CommandExecutor("TestContext", outputStream, errorStream);
executor.AddCommand<OutputCommand>();
executor.Execute("Output 'hello world'");

Will write to the outputStream:

hello world

Example 2
The following example will create a CommandExecutor, register a command called "Output" and execute the command from it's child CommandExecutor.

TextWriter outputStream = new StringWriter();
TextWriter errorStream = new StringWriter();

CommandExecutor parentExecutor = new CommandExecutor("TestContext", outputStream, errorStream);
CommandExecutor childExecutor = new CommandExecutor("ChildContext", parentExecutor);
parentExecutor.AddCommand<OutputCommand>();
childExecutor.Execute("Output 'hello world'");

Will write to the outputStream:

hello world
